Areas de Git

Working Directory

El area de trabajo corresponde a la carpeta donde iniciamos el repositorio y tambien donde vamos a estar realizando los cambios de nuestro proyecto. Practicamente no necesitamos utilizar ningun comando, salvo en muy pocas situaciones.

Staging Area (INDEX)

El area de control de cambios se utiliza para agregar los archivos a los que queremos realizarles seguimiento, de esta manera GIT nos puede notificar cuando estos fueron modificados y el estado de los mismos; para lograr esto, se realiza un Snapshot (captura de pantalla) del codigo del archivo. Los archivos que no sean agregados al INDEX, no podran ser reestablecidos y tampoco se llevara contro de los mismos.

Repository

Corresponde al lugar donde se almacenan las capturas en formato BLOB (Binary Large Object File) generando un numero de HASH utilizando la informacion de nombre usuario, correo, titulo del mensaje, registro de cambios, ademas de la fecha y la hora en la que se realiza la confirmacion. Esto es necesario para registrar la relacion que existe entre dicho commit y las confirmaciones de cambios anteriores.
